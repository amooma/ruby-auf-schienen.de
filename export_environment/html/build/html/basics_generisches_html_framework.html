3.5. Generisches HTML-Framework (Layout)

Bei allen bisherigen mit erb gerenderten HTML-Seiten fehlten wichtige HTML-Elemente, die die Seite syntakisch erst korrekt machen. Heute Webbrowser stellen die Seiten trotzdem dar, aber auch nur deshalb weil sie so fehlertollerant programmiert wurden. Bis jetzt wurde von erb immer nur der Inhalts-Kern der Seite gerendert und an den Browser ausgeliefert. Hier der Code der http://0.0.0.0:3000/example/test Seite:
<h1>Example#test</h1> 
<p>Find me in app/views/example/test.html.erb</p> 
 
<p> 
Die aktuelle Uhrzeit ist 
 
</p> 
 
<ul> 
<li>1</li> 
<li>2</li> 
<li>3</li> 
<li>4</li> 
<li>5</li> 
</ul>
Für das Gerüst drum rum müssen wir die Datei app/views/layouts/application.html.erb mit einem Editor erstellen und beispielhaft wie folgt befüllen:
<!DOCTYPE html>
<html>
<head>
  <title>Unsere Filmdatenbank</title>
</head>
<body>

<%= yield %>

</body>
</html>
Wenn wir danach die http://0.0.0.0:3000/example/test Seite noch einmal aufrufen, bekommen wir folgenden HTML-Code ausgeliefert:
<!DOCTYPE html> 
<html> 
<head> 
  <title>Unsere Filmdatenbank</title> 
</head> 
<body> 
 
<h1>Example#test</h1> 
<p>Find me in app/views/example/test.html.erb</p> 
 
<p> 
Die aktuelle Uhrzeit ist 
 
</p> 
 
<ul> 
<li>1</li> 
<li>2</li> 
<li>3</li> 
<li>4</li> 
<li>5</li> 
</ul> 
 
</body> 
</html>
Das sieht doch schon direkt viel besser aus. Mit dem Befehl yield bzw. mit der erb-Zeite <%= yield %> wird der Kern der Seite in das Aussengerüst eingebettet. Ich überlasse es Ihrer Fantasie und Ihren HTML-Künsten mit der Möglichkeit zu zaubern. Natürlich müssen Sie diese Konstrukt nicht benutzen und können jede Seite im View direkt vollständig programmieren.
